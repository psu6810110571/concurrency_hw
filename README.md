# การทดลองเขียนโค้ด Concurrency ใน Python

โปรเจกต์นี้เป็นการทดลองเปรียบเทียบการทำงานของ `threading`, `asyncio` และ `Process Pool` ตามโจทย์ที่ได้รับมอบหมายครับ

## โครงสร้างของโปรแกรม
โปรแกรมถูกจัดโครงสร้างโดยแยกไฟล์ตามวิธีการทำ Concurrency เพื่อความเป็นระเบียบและอ่านง่าย ดังนี้:

1. **`task_thread.py` (I/O Bound Task):** - ทดสอบดึงข้อมูลจากเว็บไซต์ 50 ครั้ง โดยใช้ `ThreadPoolExecutor` เหมาะกับงานที่ต้องรอเครือข่าย
2. **`task_asyncio.py` (I/O Bound Task):** - ทดสอบดึงข้อมูลจากเว็บไซต์ 50 ครั้งเช่นกัน แต่ใช้ `aiohttp` และฟีเจอร์ `async/await` เพื่อสลับการทำงานแบบไม่ต้องรอให้ดาวน์โหลดเสร็จทีละอัน
3. **`task_process.py` (CPU Bound Task):** - ทดสอบคำนวณตัวเลขทางคณิตศาสตร์ปริมาณมาก โดยใช้ `ProcessPoolExecutor` แยกการประมวลผลไปยัง CPU หลายๆ คอร์ เพื่อลดข้อจำกัดของ Python GIL (Global Interpreter Lock)
4. **`main.py`:** - เป็นไฟล์หลักที่ใช้ import ฟังก์ชันจากทั้ง 3 ไฟล์มารันต่อเนื่องกัน เพื่อเปรียบเทียบเวลาที่ใช้ในการทำงาน

## วิธีการรันโปรแกรม
1. ติดตั้งไลบรารีที่จำเป็น:
   `pip install requests aiohttp`
2. รันคำสั่งเพื่อทดสอบการทำงานทั้งหมด:
   `python main.py`
   *(หรือสามารถเลือกรันทดสอบแยกทีละไฟล์ได้ เช่น `python task_asyncio.py`)*
